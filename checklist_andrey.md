- [Lombok](https://projectlombok.org/)

- Генерация тестовых данных — [JavaFaker](https://github.com/DiUS/java-faker)
- CI — Jenkins

## CI/CD

Сборка и прогон тестов настроены через `Jenkinsfile`:

- checkout проекта из репозитория `master`;
- параметризованный запуск (`BROWSER`: chrome / firefox / edge, `TEST_USER`, `TEST_PASSWORD`);
- `mvn clean test` в headless-режиме;
- публикация результатов JUnit и Allure-отчёта после каждого прогона.

Кросс-браузерное тестирование реализовано через параметр `browser` и suite-файл (`YakushinTests.xml`).

## Структура проекта

```
src/main/java/api/adapters   — адаптеры для работы с API (User, Car, House, Login)
src/main/java/api/models     — модели запросов/ответов
src/main/java/db             — подключения к БД (User, Car, House)
src/main/java/ui/pages       — Page Object классы
src/main/java/ui/dto         — DTO для UI-форм
src/main/java/ui/wrappers    — обёртки над UI-элементами (Input, DropDown, RadioButton)
src/test/java/tests/ui       — UI-тесты
src/test/java/tests/api      — API-тесты
src/test/java/tests/jdbc     — JDBC-тесты
src/test/java/listners       — TestListener, AnnotationTransformer
src/test/java/utils          — Retry-механизм
```

## Обнаруженные баги

| № | Описание                                                                                      |
|---|-----------------------------------------------------------------------------------------------|
| 1 | Продажа отсутствующей у пользователя машины — сервер возвращает `200` вместо ожидаемого `404` |

## Запуск проекта

```bash
mvn clean test -Dheadless=true -Dbrowser=chrome -Dtest.user=user@pflb.ru -Dtest.password=***
```

Прогон через Jenkins — параметризованная джоба (`BROWSER`, `TEST_USER`, `TEST_PASSWORD`), результаты публикуются в JUnit
и Allure.

> Доступы к тестовому полигону (веб-интерфейс, read-only пользователь БД) намеренно не публикуются в README — они
> переданы команде отдельно в задании курса.
